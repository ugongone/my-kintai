## Git Commit メッセージ
- Git Commitメッセージは日本語で書くこと
  - 例) feat: ~の機能を追加
- **Subject** (1行目): 50文字以内で、変更の要約を簡潔に書く。末尾にピリオドは不要。
- **Body** (3行目以降): 変更の「理由（Why）」と「内容（What）」を具体的に説明する。「どのように（How）」はコードを見ればわかるため、重要でない限り省略する。箇条書き（-）を使用しても良い。

###  Allowed Types
- **feat**: 新機能の追加
- **fix**: バグ修正
- **docs**: ドキュメントのみの変更
- **style**: コードの動作に影響しない変更（空白、フォーマット、セミコロン欠落など）
- **refactor**: バグ修正も機能追加も行わないコード変更（リファクタリング）
- **perf**: パフォーマンスを向上させるコード変更
- **test**: テストの追加や既存テストの修正
- **chore**: ビルドプロセスやドキュメント生成などの補助ツールやライブラリの変更
