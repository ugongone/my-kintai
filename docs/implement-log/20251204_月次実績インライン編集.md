## 概要

月次実績ページの編集UIをトグル展開式からテーブル上のインライン編集に変更した。
編集ボタンを押すと、追加フォーム(AddEntryRow)と同様のUIが該当行に表示され、その場で編集できるようになった。

## 実装内容

### 1. EditEntryRowコンポーネントの新規作成
- ファイル: `src/components/history/EditEntryRow.tsx`
- AddEntryRowと同様のインライン編集UIを提供
- 既存のDailyStatから初期値を自動取得（開始時刻、終了時刻、休憩時間、翌日フラグ）
- 編集用に背景色をamber系に変更（追加はblue系）

### 2. HistoryTableの修正
- ファイル: `src/components/history/HistoryTable.tsx`
- トグル展開ロジック（expandedDates, toggleExpand）を削除
- 不要なインポート（useState, Badge, ChevronDown, ChevronRight）を削除
- propsを変更:
  - `onEditEntry` → `onEditStat`（TimeEntry単位からDailyStat単位へ）
  - `onDeleteEntry` → `onDeleteStat`
  - 新規追加: `onSaveEdit`, `onCancelEdit`, `editingDate`
- 編集中の行は`EditEntryRow`コンポーネントを表示

### 3. history/page.tsxの修正
- ファイル: `src/app/(authenticated)/history/page.tsx`
- ManualEntryModalを削除（インライン編集に置き換え）
- 状態管理を変更:
  - `editingEntry` → `editingDate`（編集中の日付を管理）
  - `deletingEntry` → `deletingStat`（削除対象を日単位に変更）
- `replaceEntriesForDate`を使用して日単位で打刻を置換

### 4. useTimeEntriesフックの拡張
- ファイル: `src/hooks/useTimeEntries.ts`
- `replaceEntriesForDate`関数を追加
  - 指定日の既存エントリを全削除
  - 新しいエントリを追加（addMultipleEntriesを再利用）

## Q&A

### Q1: なぜTimeEntry単位ではなくDailyStat単位で編集するように変更したのか？
A: AddEntryRowと同じUIを使用するため。AddEntryRowは1日分の勤務データ（開始・終了・休憩）を一括で入力する設計になっているため、編集も同様に日単位で行うことでUIの一貫性を保つ。

### Q2: replaceEntriesForDateはなぜ削除→追加の2段階で処理しているのか？
A: 編集では時刻だけでなく、休憩の有無なども変更できる。既存エントリを個別に更新するよりも、全て削除して新規追加する方がシンプルで、addMultipleEntriesのロジックを再利用できる。

### Q3: 編集行と追加行の背景色が異なる（amber vs blue）のはなぜ？
A: ユーザーが「今何をしているか」を視覚的に区別しやすくするため。青は追加（新規）、オレンジ/琥珀は編集（既存の変更）というUIの慣例に従っている。
