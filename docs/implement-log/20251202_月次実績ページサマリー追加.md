## 概要

デザインモック（`design/designMock.jsx`）の更新に伴い、月次稼働実績ページにサマリーカードを追加し、打刻追加ボタンの位置をヘッダー内に移動した。

## 実装内容

### 1. サマリーカードの追加（月次実績ページ）
- ページ上部に2つのサマリーカードを横並びで追加
- 左カード: 総稼働時間（`Clock`アイコン付き、`bg-blue-50`背景）
- 右カード: 概算報酬額（`JapaneseYen`アイコン付き、`bg-orange-50`背景）
- 既存の`useSettings`フックから時給を取得して計算

### 2. 打刻追加ボタンの移動
- テーブル内からヘッダー領域（CSV出力ボタンの左側）へ移動
- 青いボタンスタイル（`bg-blue-600`）を適用
- `Plus`アイコンをボタン内に配置

### 3. HistoryTableコンポーネントの更新
- 打刻追加の状態管理を親コンポーネントに移動
- 新しいプロップ追加:
  - `isAddingEntry?: boolean` - 追加行の表示状態
  - `onCancelAdd?: () => void` - 追加キャンセル時のコールバック
- 内部の追加ボタンを削除し、親から制御される形式に変更

### 4. AddEntryRowのデザインモック準拠
- `colSpan`構造から各列に対応する`td`構造に変更
- 入力フィールドスタイル: `border border-blue-300 rounded px-2 py-1 text-slate-800 text-xs`
- ステータス列: 「稼働」バッジを固定表示
- 実働時間列: 「自動計算」テキスト表示
- 休憩: 開始・終了の2フィールドを縦に配置
- 操作ボタン: `Save`アイコン（青）と`X`アイコン（白枠）に変更
- アニメーション: `animate-fade-in`追加
- カラーパレット: `gray-*` → `slate-*`に統一

## Q&A

### Q1: なぜサマリーカードをダッシュボードとは別のスタイルにしたのか？
A: ダッシュボードのSummaryCardは1カード縦積み構造だが、月次実績ページではシンプルに2カード横並びにして、必要最小限の情報（時間と報酬）のみを表示している。これは、月次実績ページのメインコンテンツがテーブルであり、サマリーは補助的な情報として控えめに表示するため。

### Q2: 打刻追加ボタンをヘッダーに移動した理由は？
A: デザインモックに準拠するため。また、ボタンをヘッダーに配置することで、テーブルのデータがない状態でもユーザーが打刻を追加できることが視覚的に明確になる。

### Q3: HistoryTableの状態管理を親に移動した設計上の利点は？
A: 「状態の持ち上げ（lifting state up）」パターンを適用。これにより、親コンポーネントが追加ボタンを管理でき、テーブルコンポーネントは表示に専念できる。また、ボタンの位置を柔軟に変更できるようになった。

### Q4: AddEntryRowでcolSpan構造からtd構造に変更した理由は？
A: デザインモックではテーブルの各列に対応する形で入力フィールドが配置されている。これにより、既存データの行と入力行の列が揃い、視覚的な一貫性が保たれる。

### Q5: 休憩入力を2フィールド（開始・終了）のまま維持した理由は？
A: デザインモックでは休憩が単一フィールドだが、現在の実装では休憩開始・終了時刻を個別に管理している。機能的な一貫性を優先し、2フィールド入力を維持しつつ、縦に配置してスペースを節約した。
